# ColorWars Duel PWA

Лёгкий браузерный PWA-прототип ColorWars: человек играет против компьютера за контроль над цветными квадратными плитками.

## Почему PWA подходит для MVP

Для первого прототипа достаточно статической PWA-архитектуры:

- нет сервера и сборщика, игру можно открыть через любой статический хостинг;
- `index.html` содержит UI, правила, игровое состояние и рендер;
- `manifest.webmanifest` делает приложение устанавливаемым;
- `sw.js` кэширует app shell для офлайн-запуска;
- прогресс матча и статистика сохраняются в `localStorage`.

Такой подход хорошо подходит для проверки core loop: выбор цвета -> захват соседних плиток -> ход компьютера -> проверка победы.

## Как запустить

Нужен любой локальный HTTP-сервер, потому что service worker не работает с прямым открытием файла.

```bash
python3 -m http.server 4173
```

Затем откройте:

```text
http://localhost:4173
```

## Текущая механика

- начальный экран содержит меню: `Старт`, `Настройки`, `Правила`;
- по кнопке `Старт` генерируется поле и начинается новая игра;
- в настройках можно выбрать количество плиток по горизонтали и вертикали от 6 до 24;
- размер плиток автоматически подстраивается под выбранные параметры поля;
- 2 игрока: человек и компьютер;
- цвета плиток: жёлтый `#f5f45e`, синий `#0101ff`, оранжевый `#fe9e2e`, голубой `#7cbce2`, зелёный `#007701`, красный `#db0102`, серый `#7b7b7b`;
- на старте у каждого игрока есть случайное количество связанных плиток своего случайного цвета;
- все соседние со стартом плитки этого же цвета сразу становятся территорией игрока;
- человек стартует в левом нижнем углу, компьютер - в правом верхнем;
- первый ход делает человек;
- каждый ход игрок выбирает новый цвет через компактную нижнюю палитру в одну строку или тапом по плитке такого цвета на поле;
- нельзя выбрать свой текущий цвет или текущий цвет соперника;
- все плитки игрока перекрашиваются в выбранный цвет;
- соседние сверху, снизу, справа и слева плитки такого же цвета становятся плитками игрока;
- если область замкнута плитками игрока, она захватывается целиком независимо от цветов внутри;
- край игрового поля может быть частью замыкания: линия от края до другого края или до этого же края тоже может закрыть область;
- игра заканчивается, когда один из соперников захватывает больше 50% поля;
- при победе человека показывается салют, при поражении - отдельная анимация проигрыша;
- на финальном экране доступны кнопки `Новая игра` и `Переиграть`;
- во время игры и на финальном экране можно вернуться в главное меню;
- кнопка `Откат` возвращает к предыдущему ходу человека, последовательно вплоть до начала партии;
- кнопка `Сначала на этом поле` мгновенно возвращает партию к начальному состоянию без генерации нового поля;
- кнопка `Новое поле` создаёт новую случайную карту.

Состояние текущей партии, стартовое поле, история откатов и настройки размера поля сохраняются в `localStorage`.
