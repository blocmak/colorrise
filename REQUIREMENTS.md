# Требования к ColorRise

## Общая идея

- Игра создаётся как PWA-приложение.
- Игра должна запускаться в браузере и подходить для запуска на смартфоне.
- Игра должна размещаться на HTTPS-хостинге, в том числе GitHub Pages.
- На начальном экране игра называется `ColorRise`.

## Игровое поле

- Игровое поле состоит из цветных плиток квадратной формы.
- Количество плиток по горизонтали и вертикали задаётся в настройках.
- Размер плиток должен автоматически подстраиваться под выбранные параметры поля и размер экрана.
- Расстояние между плитками должно быть небольшим.
- Радиус скругления плиток должен быть небольшим.
- Захваченные плитки не должны дополнительно выделяться:
  - без контура;
  - без свечения;
  - без масштабирования;
  - без надписей `Я` и `AI`.
- Плитки должны отличаться только цветом.

## Цвета

В игре используется 7 цветов:

- Жёлтый: `#f5f45e`
- Синий: `#0101ff`
- Оранжевый: `#fe9e2e`
- Голубой: `#7cbce2`
- Зелёный: `#007701`
- Красный: `#db0102`
- Серый: `#7b7b7b`

## Участники

- В игре участвуют 2 игрока:
  - человек;
  - компьютер.
- Человек начинает в левом нижнем углу.
- Компьютер начинает в правом верхнем углу.
- Первый ход делает человек.
- Игроки ходят по очереди.

## Старт партии

- В начале игры каждому игроку случайно присваивается несколько связанных плиток одного цвета.
- Все соседние плитки этого же цвета, примыкающие к стартовой области, сразу становятся территорией игрока.
- Стартовое поле должно сохраняться, чтобы можно было переиграть партию на том же поле.

## Ход игрока

- Каждый ход игрок выбирает один цвет.
- После выбора цвета все плитки игрока окрашиваются в выбранный цвет.
- Соседние плитки такого же цвета становятся плитками игрока.
- Соседними считаются плитки:
  - сверху;
  - снизу;
  - слева;
  - справа.
- Диагональные плитки соседними не считаются.
- Нельзя выбрать свой текущий цвет.
- Нельзя выбрать текущий цвет соперника.
- Цвет можно выбрать двумя способами:
  - через нижнюю строку выбора цвета;
  - тапом по плитке нужного цвета на игровом поле.
- Если игрок тапнул по запрещённому цвету, ход не должен выполняться.

## Нижний выбор цвета

- Выбор цвета должен располагаться снизу под игровым полем.
- Над нижней строкой выбора цвета не должно быть текста `Выбор цвета`.
- Над нижней строкой выбора цвета не должно быть подсказки `Нажми цвет...`.
- Пространство должно использоваться преимущественно для игрового поля.
- Все 7 цветовых кнопок должны всегда помещаться в одну строку.
- Цветовые кнопки должны автоматически масштабироваться под размер экрана.

## Захват территории

- При смене цвета игрок присоединяет соседние плитки выбранного цвета.
- Если игрок окружает замкнутую область, вся область становится его территорией.
- Цвет плиток внутри окружённой области не имеет значения.
- Край игрового поля может быть частью замыкания.
- Замыкание может идти:
  - от одного края поля до другого края;
  - от края поля до этого же края.
- В таких случаях вся отделённая область должна становиться территорией игрока.

## Условие победы

- Игра заканчивается, когда один из соперников захватывает больше 50% территории.
- При победе человека должно появляться финальное сообщение с анимацией салюта.
- При проигрыше человека должно появляться финальное сообщение с анимацией проигрыша.
- В процессе игры не нужно показывать сообщения о количестве захваченных плиток.

## Откат и повтор партии

- В игре должна быть возможность хода назад.
- Откат должен возвращать игру к предыдущему ходу человека.
- Откат можно активировать несколько раз подряд.
- Последовательным откатом можно вернуться к началу игры.
- Должна быть возможность переиграть игру сначала на том же поле.
- Переигрывание на том же поле должно быть мгновенным возвратом к начальному состоянию текущей партии.

## Начальный экран и меню

- При открытии приложения должен отображаться начальный экран.
- На начальном экране должно быть меню из 3 пунктов:
  - `Старт`;
  - `Настройки`;
  - `Правила`.
- По кнопке `Старт` должно появляться игровое поле и начинаться новая игра.
- По кнопке `Настройки` должно открываться окно настроек.
- По кнопке `Правила` должно открываться окно с описанием правил игры.
- В процессе игры должна быть возможность вернуться на начальный экран.

## Настройки

- В окне настроек пока должно быть 2 настройки:
  - количество плиток по горизонтали;
  - количество плиток по вертикали.
- После изменения настроек новая игра должна создаваться с выбранным размером поля.
- Настройки размера поля должны сохраняться локально.

## Экран правил

- Окно правил должно описывать основные правила игры:
  - кто играет;
  - как выбирается цвет;
  - какие цвета выбирать нельзя;
  - как захватываются соседние плитки;
  - как работает окружение;
  - когда наступает победа.

## Финальный экран

- На сообщении о победе или проигрыше должны быть кнопки:
  - `Новая игра`;
  - `Переиграть`;
  - `Меню`.
- `Новая игра` должна генерировать новое поле.
- `Переиграть` должна начинать игру заново на этом же поле.
- `Меню` должна возвращать на начальный экран.

## PWA и размещение

- Приложение должно иметь PWA-архитектуру.
- Должны быть PWA-файлы:
  - `manifest.webmanifest`;
  - `sw.js`;
  - иконка приложения.
- Приложение должно работать через HTTPS.
- Для GitHub Pages пути к PWA-файлам должны быть относительными, чтобы приложение работало из подпапки репозитория.
- Service worker должен кэшировать приложение для офлайн-запуска.
- При изменении важных файлов нужно обновлять версию cache service worker.

## Локальное хранение

- Состояние партии должно сохраняться локально.
- Стартовое поле должно сохраняться локально.
- История откатов должна сохраняться локально.
- Настройки размера поля должны сохраняться локально.
